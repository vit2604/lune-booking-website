# Production Test Checklist

Run this after frontend, backend, and PostgreSQL are deployed.

## Backend

- [ ] `GET /api/health` returns `databaseConnected: true`.
- [ ] `GET /api/rooms` returns active rooms.
- [ ] `GET /api/payment-methods` returns enabled methods.
- [ ] `POST /api/auth/admin/login` returns a JWT token.
- [ ] Admin API rejects requests without Bearer token.
- [ ] CORS allows only the production frontend domain.

## Guest Booking

- [ ] Guest can open Home.
- [ ] Guest can open Rooms.
- [ ] Guest can open Room Detail.
- [ ] Guest cannot select check-in in the past.
- [ ] Guest cannot select checkout before or equal check-in.
- [ ] Guest cannot book above room capacity.
- [ ] Guest can submit booking through `POST /api/bookings`.
- [ ] Backend calculates nights, subtotal, discount, fee, tax, and total.
- [ ] Booking code is generated by backend.
- [ ] Booking appears in PostgreSQL.
- [ ] Duplicate overlapping booking is blocked.
- [ ] Back-to-back booking where new check-in equals old checkout is allowed.

## Payment MVP

- [ ] Pay at property sets payment status to `PAY_AT_PROPERTY`.
- [ ] Bank transfer sets payment status to `PENDING`.
- [ ] VietQR/manual placeholder does not ask for real card data.
- [ ] Bank name, account number, account holder, QR URL come from backend payment settings.
- [ ] Copy account number works.
- [ ] Copy transfer content works.

## Admin

- [ ] `/admin/login` works with production admin env account.
- [ ] `/admin/dashboard` loads database stats/bookings.
- [ ] `/admin/bookings` shows real bookings from database.
- [ ] Admin can mark booking confirmed.
- [ ] Admin can mark booking cancelled.
- [ ] Admin can mark payment paid.
- [ ] Admin can mark payment pending.
- [ ] Admin can edit payment settings.
- [ ] Frontend token is stored in localStorage as admin token.
- [ ] Logout clears admin token.

## Frontend Routing

- [ ] Refresh `/rooms` does not 404.
- [ ] Refresh `/rooms/:slug` does not 404.
- [ ] Refresh `/booking` does not 404.
- [ ] Refresh `/payment` does not 404.
- [ ] Refresh `/booking-success` does not 404.
- [ ] Refresh `/admin/login` does not 404.
- [ ] Refresh `/admin/dashboard` does not 404.

## Chat / Socket

- [ ] Chat widget opens.
- [ ] Guest can send message.
- [ ] Admin Messages page can read conversations.
- [ ] Socket connection uses `VITE_SOCKET_URL`.
- [ ] If socket is unavailable, UI does not crash.

## Mobile

- [ ] Booking flow works at 375px.
- [ ] Payment method cards are easy to tap.
- [ ] Contact map buttons are easy to tap.
- [ ] Admin booking table can scroll horizontally.
